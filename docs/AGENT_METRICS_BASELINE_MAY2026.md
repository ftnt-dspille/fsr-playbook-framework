## Phase-2 tool audit findings & fixes landed (2026-05-07)

### `validate_yaml` (25 calls) — no changes needed
- Error envelope `_err(code, message, next_fix)` is the template all other tools should follow.

### `find_operation` (15 calls)
- p95 payload 8,096 chars due to high default limit. **Fix landed:** default `limit` lowered 20→10.

### `get_step_type` (12 calls)
- Returned bare `{"error": "..."}` on miss — no structured code or near-match. **Fix landed:** now uses `_err()` with `near=` close-matches against `_SHORT_TO_CANONICAL`.

### `get_op_schema` (8 calls — heaviest payload)
- Median 5,406 chars; returned all output schema blobs. `output_schema_observed` was NULL on all 6,773 ops.
- No `verbose` switch; error envelope was bare string.
- **Fixes landed:** `verbose=False` slim mode (1,250 chars vs 3,280 verbose); `_err()` envelope distinguishing `connector_not_found` from op-level `not_found`; `near=` hints from `find_connector` logic.

### `find_connector` (9 calls) — no changes needed
- Already follows the `{matches, suggestion, near}` pattern.

### Cross-cutting
- **Error envelope inconsistency** resolved: all not-found cases now use `_err()`.
- `get_op_schema` `connector_not_found` code directly addresses the 22% Phase-1C "find_connector before get_op_schema" violation rate.
- After fixes, all 216 unit tests still pass.
